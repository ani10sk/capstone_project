1. git clone <git_url>
Clones the repository onto the described location

2. git add .
Addes the files to the present commit session, the '.' adds al the files to be committed, 
optionally instead of '.' we can use certain file names also to commit only those file names.

3. git commit -m "<commit message>"
Once we have added the files required to be committed, we can commit those changes with a commit 
message

4. git push -u origin <branch_name>
Once we have committed the changes, we can push it to whichever branch we need the changes to be pushed 
into.

5. git branch <branch_name>
Creates a new branch which is based on the current branch we are curently in.

6. git checkout <branch_name>
This is used to navigate to a branch which we created previously.

7. git checkout -b <branch_name>
This is used to navigate to a new branch, and if the branch name does not exist, it creates a new branch 
with that name and then uses the same branch.
